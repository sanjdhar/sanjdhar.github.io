üìä Common Commands

Status and information

git status
Check repository status
git log --oneline
View commit history (compact)
git diff
Show unstaged changes

Staging and committing

git add .
Stage all changes
git add filename.txt
Stage specific file
git commit -m "Descriptive commit message"
Commit staged changes
git commit -am "Message"
Stage and commit all tracked files

üîÑ Undoing Changes

git reset HEAD~1
Undo last commit (keep changes)
git reset --hard HEAD~1
Undo last commit (discard changes)
git checkout -- filename.txt
Discard changes in specific file
git stash
Temporarily save changes
git stash pop
Restore stashed changes

üí° Pro Tips

Use meaningful commit messages: Instead of "fix bug", use "Fix login validation error for empty email field"

Commit often: Make small, logical commits rather than large ones

Pull before push: Always pull the latest changes before pushing your work

Use .gitignore: Add files you don't want to track (node_modules, .env, etc.)

‚ö†Ô∏è Important Notes

Never force push to shared branches: Use git push --force only on your own feature branches

Double-check before reset --hard: This permanently deletes your changes

Keep your main branch clean: Always work on feature branches and use pull requests
